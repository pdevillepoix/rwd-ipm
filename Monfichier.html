Viewport & Media Queries

Mes choix techniques dans l'utilisation des directives viewport et media queries — M2 Ingénierie Pédagogique Multimodale

Mes 4 choix de code commentés

Cette page présente quatre extraits de code issus de ce site. Chaque extrait est accompagné d'une justification de ses paramètres, en lien avec les notions vues en cours (D. Bouchard, Web Mobile — Responsive Web Design).

Choix n°1 — La directive viewport (cours §3.3) 

<meta name="viewport"
  content="width=device-width, initial-scale=1.0" />

Fonction : Cette balise, placée dans le <head>, indique au navigateur mobile comment adapter la largeur d'affichage à l'écran réel.

width=device-width : la largeur du viewport CSS devient égale à la largeur physique de l'écran. Sans ce paramètre, le navigateur simule un écran virtuel de ~980px puis réduit l'ensemble, rendant les textes illisibles et les media queries inactives.

initial-scale=1.0 : le niveau de zoom est fixé à 100 % au chargement de la page, sans agrandissement automatique.

Comme le souligne le cours (§3.5) : cette balise ne rend pas un site responsive à elle seule, elle fournit simplement au navigateur les informations nécessaires pour appliquer correctement les règles CSS responsives.

Choix n°2 — Media query mobile (cours §5.1) 

@media screen and (max-width: 480px) {
  .nav__inner {
    flex-direction: column;
  }
  .deux_colonnes {
    flex-direction: column;
  }
  .hero__titre {
    font-size: 1.6rem;
  }
}

Fonction : Cette règle s'applique lorsque la largeur de l'écran est inférieure ou égale à 480px, c'est-à-dire sur les smartphones (320 à 480px selon le tableau du cours §1.1).

screen : le type de média ciblé est l'écran, à l'exclusion des médias d'impression (cours §4.2).

max-width: 480px : la condition porte sur la largeur maximale ; les règles contenues ne s'appliquent que si la fenêtre est égale ou plus étroite que ce seuil (cours §5.1).

Les adaptations produites correspondent à la stratégie d'empilement vertical décrite au §7.2 du cours : les colonnes passent en disposition verticale pour conserver la lisibilité sur petit écran.

Choix n°3 — Media query tablette et bureau (cours §5.2) 

@media screen and (min-width: 768px) {
  .deux_colonnes {
    flex-direction: row;
  }
  .hero__titre {
    font-size: 2.8rem;
  }
}

Fonction : Cette règle s'active à partir de 768px de largeur, soit le seuil correspondant aux tablettes dans le tableau du cours (768 à 1024px, §1.1).

min-width: 768px : la condition porte sur la largeur minimale ; les règles s'appliquent dès que la fenêtre atteint ou dépasse ce seuil (cours §5.2). C'est l'approche inverse du max-width : on enrichit l'affichage au lieu de le simplifier.

La mise en page passe en deux colonnes côte à côte, ce qui exploite l'espace horizontal disponible sur tablette et bureau. Ce breakpoint à 768px est l'un des "exemples classiques" cités dans le cours (§6.1 : 480px, 768px, 1024px, 1280px).

Choix n°4 — Dimensions fluides (cours §2.2.1) 

.conteneur {
  width: 100%;
  max-width: 960px;
  margin: 0 auto;
}

img {
  max-width: 100%;
  height: auto;
}

Fonction : Ce sont les deux premiers piliers du Responsive Web Design tels que définis dans le cours (§2.2.1 et §2.2.2).

width: 100% : la largeur du conteneur s'adapte à l'espace disponible, plutôt que d'être fixée en pixels (on remplace le width: 960px figé décrit au §1.1 du cours).

max-width: 960px : on fixe néanmoins un plafond pour éviter que les lignes de texte deviennent trop longues sur de très grands écrans.

max-width: 100% / height: auto sur les images : une image ne dépasse jamais son conteneur, et ses proportions sont conservées automatiquement. C'est l'exemple exact du cours (§2.2.2).

Le responsive en pratique

Un même site, consulté sur des appareils différents, doit offrir une expérience lisible et fonctionnelle à chaque fois. C'est l'objectif du Responsive Web Design : une seule base de code, plusieurs contextes d'affichage (cours §2.1).

En ingénierie pédagogique multimodale, cette exigence est directement liée à l'accessibilité des dispositifs de formation numérique. Un apprenant peut accéder à une ressource depuis un ordinateur de bureau, une tablette ou un smartphone — l'interface doit s'adapter sans qu'une version spécifique soit nécessaire.
